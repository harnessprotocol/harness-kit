v0.3.0 ships three new workflow plugins, Harness Protocol v1 alignment for harness-share, and an eval framework for measuring skill quality across models.

<!-- truncate -->

## What's new

### open-pr and merge-pr

Two focused PR workflow plugins replacing the former `ship-pr` monolith.

`/open-pr` handles the front half: run local tests, open a PR with a structured description, run a code review via the `review` plugin (flags MUST FIX items before proceeding), and check CI with quick-fix attempts for simple failures.

`/merge-pr` handles the back half: verify CI and review state, sync with the base branch (rebases if behind), confirm with you, then squash merge and clean up. Works on any open PR — it doesn't require `/open-pr` to have created it.

The split lets you invoke each step independently. Use `/open-pr` when you're done coding, review the PR, then `/merge-pr` when it's time to land it.

### pull-request-sweep

`/pull-request-sweep` sweeps all your open PRs across repos in one pass. It discovers every non-draft PR, enriches each with CI status and merge state, triages them into categories (MERGE_READY, NEEDS_REVIEW, BEHIND_BASE, CI_FAILING, CHANGES_REQUESTED, CONFLICTING), shows you the table before doing anything, then acts: merges what's ready, rebases what's behind, fixes simple CI failures (lint, format, typo), and runs parallel code reviews on unreviewed PRs. Produces a full report at the end.

Pairs with `/loop 30m /pull-request-sweep` for ongoing PR babysitting.

### harness-share — Harness Protocol v1

`/harness-export` now writes [Harness Protocol v1](https://harnessprotocol.io) format — `version: "1"` (string), `source: owner/repo` per plugin, and a `metadata:` profile block. `/harness-import` handles both the new format and legacy configs. A new `/harness-validate` skill validates any `harness.yaml` against the Protocol v1 JSON Schema, reporting field errors, unknown keys, and missing required sections.

### Eval framework

The eval framework measures skill quality across models. Tasks are defined in YAML with structure checks (required sections, word count, must-contain patterns) and model-graded rubric criteria. Official evals live in `evals/tasks/`; plugin authors can ship their own alongside their skills (beta). See the [writing evals guide](/docs/guides/writing-evals).

### Docs

New concept pages: [harness-kit vs A2A vs Claude Agent SDK](/docs/concepts/comparison) — when to use each, how they stack, and how they complement each other. New [Harness Protocol overview](/docs/concepts/harness-protocol).

## All plugins

| Plugin | Invocation | What it does |
|--------|-----------|--------------|
| explain | `/explain` | Layered code explanations with 6-section structure |
| research | `/research` | Process any source into a structured knowledge base |
| data-lineage | `/data-lineage` | Column-level lineage tracing with SVG diagrams |
| orient | `/orient` | Topic-focused session orientation |
| capture | `/capture` | Session information capture for daily reflection |
| review | `/review` | Structured code review with severity labels |
| docgen | `/docgen` | Generate or update README, API docs, architecture, changelog |
| harness-share | `/harness-export` · `/harness-import` · `/harness-validate` | Export, import, and validate harness configuration |
| open-pr | `/open-pr` | Pre-flight checks and PR creation |
| merge-pr | `/merge-pr` | Verify CI, sync base, squash merge |
| pull-request-sweep | `/pull-request-sweep` | Cross-repo PR sweep and merge |

## Install

```
/plugin marketplace add harnessprotocol/harness-kit
/plugin install open-pr@harness-kit
/plugin install merge-pr@harness-kit
/plugin install pull-request-sweep@harness-kit
```
